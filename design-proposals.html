Portfolio Redesign v2

Creative Direction

Three approaches, all minimal, all alive

Each option leans into what makes your site unique: you already have a stunning Three.js scene that most portfolio sites can't touch. The question is how to extend that creative energy across the whole site, not just the gallery page. No photos, no stat bars, no hero templates. Just craft.

Design principles (shared across all options)

01 GENERATIVE > STATIC

Use procedural/noise-based visuals as backgrounds instead of flat colors or stock imagery. Every visit feels subtly different. You already have the Three.js chops — use them site-wide.

02 TYPOGRAPHY AS IDENTITY

Let type carry the personality instead of a photo. A distinctive serif for your name, monospace for metadata, clean sans for body. The type IS your avatar.

03 MOTION WITH PURPOSE

Every animation should feel intentional — page transitions via SvelteKit's view transition API, staggered reveals on route change, subtle parallax. Never gratuitous, always earned.

Option A

Generative canvas background

A full-viewport canvas runs behind the entire site — not just the gallery. A simplex noise field, particle system, or flow field that reacts subtly to scroll position and mouse. Content floats over it with a gradient veil. Each page gets a different noise seed or color palette so the background shifts as you navigate.

Recommended

Homepage: Your name over a living canvas

The grass scene you built proves you can make something beautiful. Imagine a simpler, subtler version (flow field, particles, Perlin terrain) filling the viewport on every page, with content layered on top. Navigation triggers a smooth noise-seed transition.

The canvas above is live — it's running a simplex noise flow field with particles right now. This is the kind of generative background that would fill the viewport behind your content. The gradient veil on the left keeps text readable. Each page could shift the hue or noise frequency for subtle variation.

How it works technically

A single <canvas> in +layout.svelte renders behind all content using position: fixed; z-index: -1. The animation runs in requestAnimationFrame and reads scrollY to gently influence the noise offset. SvelteKit's onNavigate triggers a color/seed transition on route change.

Performance considerations

Canvas 2D is sufficient — no WebGL needed for a flow field. Use devicePixelRatio capping at 2x. Pause animation when tab is hidden via visibilitychange. On mobile, reduce particle count or swap to a static noise texture for battery life.

Content readability

The gradient veil (background: linear-gradient(90deg, rgba(0,0,0,0.7)...)) ensures text contrast. The canvas should be low-contrast and slow-moving — it's atmosphere, not the main event. Content areas can add a subtle backdrop-filter: blur() for extra separation.

Page-specific moods

Each route can pass a reactive theme prop to the canvas component: warm earth tones for projects, cool blues for education, the existing green for gallery. The transition between them uses lerp() on the color values.

Option B

Bento grid homepage

Instead of a linear page with a hero, the home page IS a grid of cards — each one a portal to a section. Different sizes create visual hierarchy without a traditional layout. Hover reveals previews. The grid itself is the navigation.

Alternative

Homepage as spatial navigation

The intro card is largest (2x2). Other sections get cards that preview their content — skills shows mini-tags, gallery shows a noise gradient, projects shows a card preview. No separate "About" page needed — the intro card IS the about.

Grid animation on load

Cards stagger in with animation-delay on mount — each card fades up from below with a slight offset. Use Svelte's fly transition with per-card delay calculated from grid position. Takes ~600ms total for the full grid to appear.

Hover micro-interactions

Cards lift on hover with translateY(-4px) and a soft glow via box-shadow. The arrow indicator slides right. The gallery card could show a subtle noise animation on hover via a tiny canvas element.

Combines with Option A

The bento grid works great on top of a subtle generative canvas. The card backgrounds become backdrop-filter: blur(16px) glass panels over the living background. Best of both worlds — spatial navigation with generative atmosphere.

Mobile adaptation

Grid collapses to 2 columns on tablet, single column on mobile. Intro card stays full-width. Cards become tappable with a press-scale effect. The spatial feel translates naturally to vertical scrolling.

Option C

Scroll-driven single page

Instead of separate routes, everything lives on one page. Scroll snaps between full-viewport sections. Each section has its own visual treatment — different background gradients, type scales, and layout. An intersection observer triggers reveals as you scroll into each section. The left-side page indicator shows where you are.

Alternative

Vertical journey through your work

The scroll IS the navigation. A thin progress bar or section dots on the side show position. Content animates in as it enters the viewport — type slides, cards fan out, timelines draw themselves. The gallery section has the grass scene inline. It's a curated tour, not a reference document.

Intersection Observer reveals

Each section uses IntersectionObserver with threshold 0.3 to trigger entrance animations. Text slides up from below, visual elements scale in from 0.95. Use Svelte's transition:fly on visibility state change. Stagger child elements with CSS animation-delay.

Scroll-linked parallax

The visual panels on the right move at 0.8x scroll speed using transform: translateY(calc(var(--scroll) * -0.2)) where --scroll is updated via scroll event. Keep it subtle — 10-20% offset max.

Trade-offs

Single-page layouts hurt deep-linking (no direct URL for education vs experience). Mitigation: use history.replaceState to update the URL hash as sections scroll into view, so links still work. SEO is less relevant for a personal portfolio.

Gallery section inline

The grass Three.js scene becomes one of the scroll sections — it renders in a viewport-sized container within the scroll flow. Use IntersectionObserver to start/stop the animation loop when visible, saving GPU when scrolled past.

Recommendation

A + B combined

Use the generative canvas as an always-on atmospheric background (Option A), with the bento grid as the home page layout (Option B). The glass-card effect of bento cells floating over a living noise field creates something genuinely distinctive. Inner pages (education, experience, etc.) keep the canvas background but use normal content layouts with the existing sidebar-to-top-nav transition later.
